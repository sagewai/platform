# Integrations

This page links to reference examples for connecting Sagewai to the tools your team already uses: CLI proxies, the agent registry, and Harness primitives.

**Prerequisites:** Sagewai SDK installed and a running gateway. See [Get started](/docs/get-started/first-agent) if you haven't set up your first agent yet.

---

## CLI proxies

Route an existing AI CLI through Sagewai to add observability, cost tracking, and Sealed credential handling without changing the developer's day-to-day workflow.

| # | Example | Tool |
|---|---------|------|
| 09 | [proxy_claude_code](https://github.com/sagewai/platform/blob/main/packages/sdk/sagewai/examples/09_proxy_claude_code.py) | Claude Code CLI |
| 10 | [proxy_cursor](https://github.com/sagewai/platform/blob/main/packages/sdk/sagewai/examples/10_proxy_cursor.py) | Cursor IDE |
| 11 | [proxy_codex](https://github.com/sagewai/platform/blob/main/packages/sdk/sagewai/examples/11_proxy_codex.py) | OpenAI Codex |

When to use: your team is already running one of these CLIs and you want per-request cost tracking and audit logs without asking developers to change their setup.

## Agent registry

Publish agents so other teams or services can invoke them by name. Consume published agents without re-implementing the integration on the caller side.

| # | Example | Direction |
|---|---------|-----------|
| 14 | [register_agent](https://github.com/sagewai/platform/blob/main/packages/sdk/sagewai/examples/14_register_agent.py) | Publish to registry |
| 15 | [discover_agents](https://github.com/sagewai/platform/blob/main/packages/sdk/sagewai/examples/15_discover_agents.py) | Consume from registry |

When to use: you are shipping an agent that other teams should invoke without re-implementing the client integration themselves.

## Harness primitives

These examples give direct access to the proxy and agent boundary inside the Harness. They are reference-grade for power users integrating Sagewai into an existing tool surface.

| # | Example | What it adds |
|---|---------|--------------|
| 23 | [harness_proxy](https://github.com/sagewai/platform/blob/main/packages/sdk/sagewai/examples/23_harness_proxy.py) | Harness proxy primitives |
| 24 | [harness_agent](https://github.com/sagewai/platform/blob/main/packages/sdk/sagewai/examples/24_harness_agent.py) | Harness agent integration |

When to use: you need fine-grained control over how the proxy and agent interact, for example to inject custom routing logic or inspect classification decisions before they take effect.
